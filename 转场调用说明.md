# 舱门转场 · 调用说明（给负责关卡逻辑的同学）

> 一个即插即用的"舱门转场"。调用一个函数，屏幕上就会播放
> **唯美舱门「关闭 → 停顿 → 打开」的全屏动画（约 2 秒）**，用来遮挡场景切换的瞬间。

---

## 1. 资源在哪（都在 `Content/Transitions/`，内部你不用动）

| 资源 | 作用 |
|---|---|
| **`BP_SceneTransition`** | 转场控制器（Actor 蓝图），**对外暴露接口 `PlayTransition`** ← 你只跟它打交道 |
| `WBP_TransitionCover` | 全屏舱门 UI（内部实现，不用碰） |
| `Door_Left` / `Door_Right` | 门贴图（内部用，不用碰） |

---

## 2. 怎么接（3 步）

1. **摆一台实例**：把 `BP_SceneTransition` 从内容浏览器**拖进你的关卡**（位置随便，它是纯逻辑的，不影响画面）→ 保存关卡。
2. **在关卡蓝图调用**：在 `/asl/trigger` 分支（"整词拼对"那个 `Branch` 的 **true** 出口）里：
   ```
   Get Actor Of Class (选 BP_SceneTransition) → 拖出返回值 → 调用 PlayTransition
   ```
3. 编译保存，完成。

> 用 `Get Actor Of Class` 的好处：你不用我手动递引用，关卡里只要摆了一台 `BP_SceneTransition`，你就能找到并调用它。

---

## 3. 时间线（重要：场景在什么时候切）

转场总长约 **2 秒**：

| 时间 | 发生什么 |
|---|---|
| 0.0 ~ 0.7s | 门滑动**关闭**（逐渐盖住屏幕） |
| **0.7 ~ 1.3s** | 门**完全关闭**（屏幕被遮住） ← ⭐**场景切换 / 相机移动应该发生在这一段！** |
| 1.3 ~ 2.0s | 门**打开**（露出新场景） |

### ⚠️ 本轮测试的临时对接方式

我**还没做**"门关好了"的自动通知信号（`OnCovered`）。所以这一轮先用**临时定时**对接：

```
PlayTransition  →  Delay(0.7)  →  在这里做你的场景切换（相机移动 / 显示下个区域）
```

这样切换正好藏在关着的门后面，玩家看不到穿帮。

> 📌 **下一版**我会加一个 `OnCovered` 事件分发器，到时你只要
> **「绑定 OnCovered → 在里面做切换」** 就行，不用再写死 0.7 秒。
> 我做好会通知你改这一处。

---

## 4. 本轮测试目标（验证对接通不通）

1. 拼对单词时，屏幕能播出**舱门关→开**动画；
2. 配合 `Delay(0.4)` 临时切换，确认**"门关着时换场景、门开后是新场景"**看起来无缝。
